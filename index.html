InnerG Intel Framework

Human x Agent Collaboration Framework

The strongest AI systems do not replace humans. They extend human range. Vision, taste, judgment, and priority stay with the human. Structure, speed, iteration, and execution move through the agent.

Humans should bring vision and judgment. Agents should bring structure and motion.

Core Principle
Become intelligence. Then operationalize it.
Best Use
Turn rough direction into organized, high-quality execution.
InnerG Standard
Better thinking, faster execution, cleaner feedback, sharper outcomes.
01 / The Model

Clear roles create cleaner work.

Human-agent collaboration works best when each side owns the right layer of the system.

Human owns

  • Vision
  • Taste
  • Priorities
  • Ethics
  • Final approval
  • Strategic context

Agent owns

  • Speed
  • Structure
  • Research
  • Iteration
  • Drafting
  • Operational consistency
02 / The Collaboration Formula

Every strong prompt has five parts.

When humans give agents better shape up front, the output gets sharper, faster, and easier to approve.

01

Goal

What are we actually trying to achieve?

02

Context

Who is this for, what project is this tied to, and what already exists?

03

Constraints

Tone, deadline, budget, platform, boundaries, and what not to touch.

04

Definition of Good

What should success feel like? Premium, warm, trustworthy, conversion-focused, client-safe.

05

Next Action

What should happen right now? Draft, propose options, execute, summarize, or polish.

03 / Better Human Habits

Strong humans give direction with intent.

Weak

Make this better.

Strong

Make this feel more premium and trustworthy for daycare owners.

Weak

Nah.

Strong

Too generic, too techy. Make it warmer and more parent-friendly.

Use checkpoints

  • Ask for options
  • Review first pass
  • Approve final direction
  • Request polish pass

Build shared language

  • Premium
  • Client-safe
  • Street-clean
  • Conversion-first
  • Too AI
  • Too corporate
04 / Better Agent Habits

Strong agents earn trust through behavior.

Clarify the lane

Know whether the job is strategist, designer, researcher, builder, editor, or operator.

Reflect constraints through action

Do not just repeat the rules. Show them in the output.

Push back when needed

Catch weak ideas, bad priorities, bad UX, and mismatched tone before they ship.

Preserve continuity

Humans should not have to reteach the same standards every session.

05 / Best Operating Modes

Use the right collaboration mode for the job.

Mode 1: Human leads, agent executes

Best for clear tasks like site edits, design changes, content revisions, and operational support.

Mode 2: Human sets goal, agent finds path

Best for research, strategy, problem-solving, debugging, and figuring out the route.

Mode 3: Agent proposes, human judges

Best for naming, brand direction, offers, content angles, messaging, and UI concepts.

06 / Prompt Pattern

Use this structure every time.

Task: what needs to happen Audience: who this is for Goal: what result we want Tone: how it should feel Constraints: what cannot break Output: what you want back Example: Task: Build a prebuilt sales section Audience: daycare owners and childcare businesses Goal: make them want to claim the site Tone: premium, warm, trustworthy, conversion-focused Constraints: keep it simple, no generic AI vibe Output: ship the section and push live
07 / Common Failure Points

Most bad collaboration comes from fuzzy thinking, not weak tools.

Human mistakes that slow agent work

  • Being too vague
  • Changing direction without saying priorities changed
  • Hiding the real goal
  • Giving corrections with no explanation
  • Asking for perfection in the first pass
  • Not defining what success looks like

Agent mistakes that break trust

  • Sounding too robotic
  • Pretending confidence without enough signal
  • Overexplaining
  • Not asking when something is truly ambiguous
  • Making weak assumptions in high-stakes work
  • Failing to remember recurring preferences
08 / The Relationship

The goal is not useful AI. The goal is a better human operating system.

What the human should feel

  • I can think bigger because the agent handles execution.
  • I do not need to repeat myself all the time.
  • The agent understands my taste.
  • The agent catches things before they become problems.

What the agent should function like

  • Strategist when needed
  • Operator when needed
  • Mirror for ideas
  • Second brain for continuity
  • Execution partner under direction
